📚 Developer Documentation

Architecture

The Universal Compatibility Layer sits between clients and the routing engine. It consists of three components:

  1. ClientDetector — Identifies the client from User-Agent, headers, and API path
  2. ProtocolAdapter — Converts between OpenAI/Anthropic wire formats and a normalized SharedRequest
  3. CompatibilityLayer — Orchestrates detection, parsing, routing, and formatting

How Client Detection Works

The detector checks, in order:

  1. User-Agent header — Matches against known client patterns (ClaudeDesktop, cursor, Continue, etc.)
  2. Protocol headers — Checks for anthropic-version or openai-organization
  3. API path — Falls back to path-based detection (/v1/messages → Anthropic, /v1/chat/completions → OpenAI)

Detection returns a confidence score (0.0–1.0). Headers-based detection scores 0.9, path-based scores 0.7.

SharedRequest — Universal Request Format

All client requests are normalized into this format:

{
  model: string;         // Model name from client
  messages: [{           // Normalized messages
    role: 'user'|'assistant'|'system'|'tool',
    content: string | ContentBlock[]
  }];
  maxTokens?: number;
  temperature?: number;
  stream: boolean;
  client: {
    type: ClientType;    // Detected client
    protocol: ProtocolType; // Detected protocol
    userAgent: string;
    ip: string;
  };
}

Adding a New Client

  1. Add the client name to ClientType in types/api.types.ts
  2. Add a User-Agent pattern to UA_PATTERNS in compatibility/client-detector.ts
  3. If the client uses a custom protocol, add a _fromCustom() method to compatibility/protocol-adapter.ts

Configuration

No additional configuration needed. The compatibility engine auto-detects everything.

Set COMPAT_ENGINE_PORT to change the port (default: 20134).

Set GATEWAY_URL to point to the upstream gateway (default: http://127.0.0.1:20128).

Backward Compatibility

The existing gateway on port 20128 continues to work exactly as before. The compatibility layer on port 20134 is a new entry point that wraps it. Both can run simultaneously.
